UOM Souvenir Website — Project Context
======================================

Overview
--------
An e-commerce website for selling University of Mauritius (UOM) souvenirs and
merchandise — campus apparel, premium souvenirs, and university-branded
products. The goal is a professional, trustworthy, institutional-feeling
storefront with a product showcase, shopping cart, and (eventually) a full
purchase flow. Live at https://uom-souvenir-website.vercel.app.

Tech Stack
----------
Framework & Language
- Next.js 16.2.7 (App Router) — note: this is a newer Next.js with breaking
  changes vs. older versions
- React 19.2.4 (with React Compiler / babel-plugin-react-compiler enabled)
- TypeScript 5

Styling & UI
- Tailwind CSS v4 (via @tailwindcss/postcss) — utility classes only, minimal
  custom CSS
- GSAP 3 — scroll animations & page transitions
- Lucide React — SVG icons (no emojis as icons)
- Google Fonts: Geist (sans), Geist Mono, Playfair Display (serif display)
- Design direction: "Soft UI" — soft shadows, subtle depth, institutional blue
  palette (base bg #B0E0F8 pale blue, text #1E2019). Single fixed light scheme:
  no dark mode — globals.css sets color-scheme:light to force it for everyone.

Data & Backend
- Neon — serverless Postgres (@neondatabase/serverless, connection pooling)
- Drizzle ORM (drizzle-orm + drizzle-kit) — schema-first
- State: React Context (CartContext) for the cart

Tooling & Deployment
- Vercel (hosting + env vars for Neon connection string)
- Playwright (testing)
- tsx, dotenv

Current Structure
-----------------
- Routes: / (home), /shop, /cart
- Home components: Hero, BestSellers, BestSellersCard, StepsSection,
  FaqAccordion, CategoryCard
- Shop page (src/app/shop/page.tsx): self-contained — product cards rendered
  inline; no separate ProductCard component
- Layout: Navbar, Footer
- Reactbits (unused): PillNav (src/components/reactbits/) — GSAP pill navbar,
  imported nowhere

Conventions & Constraints (important for advice)
------------------------------------------------
- Server Components by default; "use client" only when needed
- Always use Next.js <Image> (never raw <img>) and <Link> for internal nav
- Tailwind utilities only — no inline styles / CSS modules unless forced
- This Next.js version differs from training data — APIs may have changed;
  verify against current Next 16 docs
- "Lazy senior dev" philosophy: prefer the simplest solution, avoid unnecessary
  dependencies/abstractions
- Accessibility: 4.5:1 contrast, visible focus states, prefers-reduced-motion,
  responsive at 375/768/1024/1440px
